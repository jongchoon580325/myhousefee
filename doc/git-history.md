# Git Commit History

## 📋 업데이트 규칙 및 가이드라인

### 문서 유지 방법
- **업데이트 시기**: 각 Phase 완료 또는 주요 기능 구현 완료 후
- **순서**: 최신 커밋부터 오래된 순서로 작성 (역시간 순)
- **구조**: Phase별로 섹션 구분, 각 섹션 내에서 최신순 정렬

### 테이블 형식
| 열 | 설명 | 예시 |
|---|------|------|
| 번호 | 해당 섹션 내 순번 | 1, 2, 3... |
| 날짜 | YYYY-MM-DD 또는 타임스탬프 | 2026-04-12 |
| 커밋번호 | Git 커밋 해시 (처음 7자) | 919429c |
| 커밋내용 | 커밋 메시지 (첫 줄) | feat: add feature |

### 작성 규칙
1. **Phase 제목 형식**: `## Phase N: [설명] (YYYY-MM-DD)`
2. **최신 커밋 우선**: 새로운 Phase는 문서 최상단에 추가
3. **커밋 메시지**: 원본 커밋 메시지의 첫 줄 사용 (간결하게)
4. **한글 포함**: 커밋 메시지에 한글이 있으면 그대로 유지

### 빠른 업데이트 명령
```bash
# 최근 20개 커밋 확인 (번호 | 해시 | 날짜 | 메시지)
git log --pretty=format:"%h | %ad | %s" --date=short -n 20

# 한 줄씩만 확인
git log --oneline -n 20
```
