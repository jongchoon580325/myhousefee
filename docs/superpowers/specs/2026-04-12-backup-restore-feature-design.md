# 데이터 백업/복구 및 전체 초기화 기능 설계

**작성일**: 2026-04-12  
**상태**: 설계 완료, 구현 준비

---

## 개요

SettingsView에 데이터 백업/복구 기능을 추가하고, Sidebar의 '전체 초기화' 기능을 설정 페이지로 이동하여 통합 관리하는 기능 구현.

### 배경
- 사용자 데이터 손실 방지를 위한 백업 기능 필요
- Sidebar의 '전체 초기화'를 더 안전한 설정 페이지로 이동
- 데이터 관리 기능을 한 곳에서 통합 관리

---

## 요구사항

### 1. SettingsView 확장

#### 1.1 백업/복구 섹션 추가 (아파트 정보 하단)

**전체 백업 기능:**
- rawSheets의 모든 데이터를 Excel(.xlsx) 파일로 내보내기
- 파일명 형식: `YYYY-MM-DD-전체관리비내역.xlsx`
- 각 월별 데이터가 별도 시트로 생성 (시트명: "2026-01", "2026-02", ...)
- 각 시트의 열 구조:
  - 연도-월 (예: "2026-01")
  - 항목 (예: "관리비")
  - 금월 (당월 금액)
  - 전월 (전월 금액)
  - 증감 (증감액)

**기간 백업 기능:**
- 사용자가 선택한 월별 범위로 Excel(.xlsx) 파일 생성
- UI: [시작 년, 월] [종료 년, 월] 드롭다운 또는 입력 필드
- 파일명 형식: `YYYY-MM-DD-YYYY-MM-DD-기간관리비내역.xlsx`
- 각 월별 데이터가 별도 시트로 생성
- 각 시트의 열 구조: 전체 백업과 동일

#### 1.2 복구 섹션

**복구 기능:**
- 전체 백업 Excel 파일(`YYYY-MM-DD-전체관리비내역.xlsx`)만 업로드 가능
- 복구 시 기존 rawSheets 데이터를 완전히 대체
- 복구 프로세스:
  1. 파일 유효성 검증 (파일명, 형식, 스키마)
  2. 파일 읽기 및 데이터 파싱
  3. Firestore에 저장
  4. Store 업데이트
  5. 성공/실패 메시지 표시

#### 1.3 위험 영역 섹션 (별도)

**전체 초기화 기능:**
- Sidebar에서 제거, 설정 페이지로 이동
- "위험 영역"이라는 별도 섹션 추가
- 모든 데이터 삭제 (rawSheets 초기화)
- 삭제 확인 대화상자 표시
- 확인 후 진행

## 기술 사양

### 2.1 파일 형식 및 스키마

**Excel 파일 구조:**
```
파일: 2026-01-15-전체관리비내역.xlsx
├─ 시트: 2026-01
│  ├─ 연도-월 | 항목 | 금월 | 전월 | 증감
│  ├─ 2026-01 | 관리비 | 100000 | 95000 | 5000
│  ├─ 2026-01 | 수도료 | 20000 | 18000 | 2000
│  └─ ...
├─ 시트: 2026-02
│  └─ ...
└─ ...
```

### 2.2 데이터 흐름

**백업:**
- rawSheets 객체 → Excel 파일 생성 → 사용자 다운로드

**복구:**
- 사용자 업로드 → 파일 검증 → 데이터 파싱 → Firestore 저장 → Store 업데이트

**초기화:**
- 사용자 확인 → rawSheets 초기화 → Firestore 삭제 → Store 업데이트

### 2.3 에러 처리

- 파일명 형식 검증
- Excel 파일 읽기 오류
- 스키마 검증 (열 구조)
- 복구 시 데이터 무결성 검증
- Firestore 저장 실패 처리
- 사용자 친화적 에러 메시지 표시

## 예상 UX 흐름

### 백업 흐름
1. 사용자가 설정 페이지 접속
2. "백업/복구" 섹션에서 "전체 백업" 또는 "기간 백업" 선택
3. 기간 백업의 경우 [시작 년, 월] [종료 년, 월] 선택
4. "백업 다운로드" 버튼 클릭
5. Excel 파일 다운로드 (파일명: YYYY-MM-DD-전체관리비내역.xlsx)

### 복구 흐름
1. 사용자가 설정 페이지의 "복구" 섹션에서 파일 선택
2. 전체 백업 파일만 선택 가능 (파일명 검증)
3. "복구" 버튼 클릭
4. 파일 검증 → 데이터 파싱 → 확인 대화상자
5. 확인 후 복구 진행 또는 취소
6. 복구 완료 메시지 표시

### 초기화 흐름
1. 사용자가 설정 페이지의 "위험 영역" 섹션 접속
2. "전체 초기화" 버튼 클릭
3. 경고 메시지 + 확인 대화상자
4. 확인 입력 (예: "초기화" 텍스트 입력 또는 체크박스)
5. 초기화 실행
6. 완료 메시지 표시
