# 이미지 업로드 - 수동 입력 및 사이드바 개선 설계

**작성일**: 2026-04-12  
**상태**: 설계 완료, 구현 준비

---

## 개요

현재 이미지 업로드 시 OCR을 통해 자동으로 판독되는 년/월을 사용자가 수동으로 입력할 수 있도록 변경하고, 사이드바의 년월별 보기를 연도별 펼침/닫힘 기능으로 개선합니다.

### 배경
- 영수증마다 년/월 표기 위치가 다르고, OCR이 누락하는 경우가 발생
- 사용자 입력으로 정확성 향상
- 사이드바에서 많은 월 목록이 최대 6개로 제한되어 과거 데이터 접근 불편

---

## 요구사항

### 1. UploadView - 수동 년/월 입력

#### 1.1 UI 변경
- **위치**: 이미지 미리보기 아래에 년/월 입력 섹션 추가
- **형식**: 직접 입력 필드 (텍스트 인풋)
  - **년도 입력**: 4자리 숫자 입력 필드 (예: 2026)
  - **월 입력**: 1-2자리 숫자 입력 필드 (예: 3 또는 03, 범위: 1-12)

#### 1.2 입력 흐름
1. **년도 입력**:
   - 4자리 숫자 입력 시 자동으로 **월 입력 필드로 포커스 이동**
   - 4자리 미만이면 포커스 이동 안 함
   - 숫자만 입력 가능 (유효성 검사)

2. **월 입력**:
   - 1-12 범위 입력만 허용
   - 엔터 키 또는 **저장 버튼 클릭**으로 입력값 확정
   - 입력 후 엔터 누르면 분석 버튼 활성화

#### 1.3 필수 입력
- 년/월 둘 다 입력하지 않으면 **분석 버튼 비활성화**
- 둘 다 유효한 값이 입력되어야만 분석 버튼 활성화
- 범위 검증: 년도는 숫자만, 월은 1-12

#### 1.3 분석 흐름 변경
- **이전**: 이미지만 선택 → 분석 → OCR로 년/월 자동 추출
- **이후**: 이미지 선택 → 년/월 선택 (필수) → 분석
- API 요청 시 사용자가 선택한 year/month를 body에 포함
- 분석 결과에 사용자 선택 년/월 표시

#### 1.4 저장 로직
- 분석 결과의 year/month는 사용자 선택값 사용
- 저장 시 `${year}-${month.padStart(2, '0')}` 형식의 yearMonth로 저장
- 저장 완료 시 `selectedYear` 업데이트 (선택된 년도로 설정)

### 2. Sidebar - 연도별 펼침/닫힘 기능

#### 2.1 UI 구조 변경

**현재 구조**:
```
2025년 월별 (selectedYear 기준)
  2025-12월
  2025-11월
  ... (최대 6개)
```

**새로운 구조**:
```
연도 및 월별 보기 (섹션 제목 추가)
  ▶ 2026 (접혀있음 - 기본값)
  ▼ 2025 (펼쳐있음 - 선택된 연도)
    • 1월
    • 2월
    • ... (오름차순, 모든 저장된 월)
  ▶ 2024
  ▶ 2023
```

#### 2.2 동작 사양

**연도 표시**:
- rawSheets에 존재하는 모든 연도 표시
- 내림차순 정렬 (최신 연도부터)

**펼침/닫힘**:
- 연도 클릭 시 토글 (펼침 ↔ 닫힘)
- 상태 표시: ▼ (펼침) / ▶ (닫힘)
- 기본값: 모든 연도 닫혀있음
- 단, `selectedYear`에 해당하는 연도는 자동 펼쳐짐

**월 정렬**:
- 펼쳤을 때 오름차순 표시 (1월 → 12월)
- 해당 연도에 저장된 모든 월 표시 (최대 6개 제한 제거)

**월 클릭**:
- rawSheet 뷰로 이동 (현재와 동일)
- `setSelectedRawSheetMonth(key)` + `setActiveView('rawSheet')`

#### 2.3 selectedYear 연동

- UploadView에서 이미지 저장 시 선택된 년도를 `selectedYear`로 업데이트
- Sidebar는 `selectedYear` 변경 감지
- 해당 연도의 폴더 자동 펼쳐짐
- 다른 연도가 선택되면 기존 펼쳐진 연도는 닫힘 (또는 다중 선택 가능 - 추후 결정)

## 상태 관리

### Store 변경사항

**추가 상태** (필요 시):
- 사이드바 펼침 상태를 추적할 수 있는 상태 (예: `expandedYears: Set<number>`)
- 또는 각 컴포넌트에서 로컬 상태로 관리

**기존 상태 활용**:
- `selectedYear`: UploadView에서 저장 시 업데이트
- `rawSheets`: 연도 목록 도출에 사용

## API 변경

### `/api/analyze` 요청 변경

**현재**:
```json
{
  "b64": "...",
  "mediaType": "image/jpeg",
  "idToken": "..."
}
```

**변경 후**:
```json
{
  "b64": "...",
  "mediaType": "image/jpeg",
  "idToken": "...",
  "year": 2026,
  "month": 3
}
```

**응답**: 변경 없음 (year/month는 요청에서 받은 값 사용)

## 예상 UX 흐름

1. **사용자가 이미지 업로드**
   - 파일 선택 → 미리보기 표시
   - 년도 입력필드에 "2026" 입력 → 자동으로 월 입력필드로 포커스 이동
   - 월 입력필드에 "3" 입력 → 엔터 키 또는 저장 버튼 클릭
   - "AI 분석 시작" 버튼 활성화

2. **분석 완료**
   - 분석 결과에 "2026년 3월" 표시
   - 항목 테이블 표시

3. **데이터 저장**
   - "데이터 저장" 클릭
   - `selectedYear` = 2026 업데이트
   - 대시보드로 이동

4. **사이드바 자동 업데이트**
   - Sidebar에서 "2026" 연도 자동 펼침
   - "3월" 항목 표시
   - 사용자가 "3월" 클릭 → rawSheet 뷰 표시

## 테스트 시나리오

- [ ] 년도 4자리 미만 입력 시 월 필드로 포커스 이동 안 함 확인
- [ ] 년도 4자리 입력 시 월 필드로 자동 포커스 이동 확인
- [ ] 월 입력: 1-12 범위만 허용, 범위 밖 입력 시 거부 확인
- [ ] 년/월 입력 없이 분석 버튼 클릭 시 비활성화 확인
- [ ] 년/월 모두 입력 후 분석 버튼 활성화 확인

- [ ] 분석 결과에 입력된 년/월 올바르게 표시
- [ ] 저장 후 selectedYear 업데이트 확인
- [ ] Sidebar에서 해당 연도 자동 펼침 확인
- [ ] 다른 연도 펼침 시 이전 연도 닫힘 (또는 다중 펼침) 확인
- [ ] 월을 클릭하면 rawSheet 뷰 이동 확인
- [ ] 모바일 화면에서도 입력/포커스 이동 정상 동작 확인
